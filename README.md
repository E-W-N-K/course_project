# course_project
Общее тз по курсачу:
Реализовать роль администратора и пользователя. Администратор может  добавлять/редактировать информацию меню,  управлять заказами и отслеживать их статус. Пользователь может просматривать предложенный вариант меню и информацию о блюдах, оформлять заказы, выбирать способ доставки и оплаты. Реализовать базу данных, в которой будет обновляться актуальная информация о блюдах, заказах и пользователях. Реализовать вход пользователя и администратора, а также валидацию проверки вводимых данных.

Нужные окна:
Окно регистрации

Пользователь:
  Окно с личной информацией аккаунта и текущими заказами (имя пользователя, адрес, заказы)
    На этих окнах + поисковая строка по названию, переход на окно резгистрации/окно с личной инфой, переход на корзину:
      Окно с выбором ресторанов (карточки ресторанов с фото, названием, адресом)
      Окно с выборо блюдом из меню выбранного ресторана (фото, название, цена, добавление в корзину) 
      Окно с блюдом (фото, название, описание, цена, добавление в корзину)
  Окно с корзиной (собранная корзина с карточками блюд (название, цена), удаление из корзины, переход к оформлению)
  Окно для оформления (способ оплаты, способ доставки, адрес) 
    при оплате картой - окно с добавлением карты
    при заказе на определённое время - окно с выбором даты и времени
  Окно с успешным оформлением заказа

Админ:
  Окно администратора с инфой о всех заказах (что заказали, куда, кто, когда, статус (в процессе/завершён)) и возможностью удалять заказы (условно, крестик для удаления)
  Окно для редактировани блюд из ресторанов (цены, удаление блюда) - такое же как и "Окно с выборо блюдом из меню выбранного ресторана (фото, название, цена, добавление в корзину)", но с крестиком на блюде для удаления и подсвеченой ценой для изменения
  Окно для добавления нового блюда (создание названия, описания, цены, добавление фото)
